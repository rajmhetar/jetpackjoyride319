Draw a raw bitmap to the RAM buffer. 


\begin{DoxyParams}{Parameters}
{\em xpos} & Horizontal position of bottom left corner of image, columns from the left edge.~\newline

 0 is on the left; 126 is near the right \\
\hline
{\em ypos} & Vertical position of bottom left corner of image, rows from the top edge.~\newline
 must be less than 64~\newline
 2 is near the top; 63 is at the bottom \\
\hline
{\em ptr} & Pointer to a 16-\/color BMP image. \\
\hline
{\em threshold} & Grayscale colors above this number make corresponding pixel \textquotesingle{}on\textquotesingle{}.~\newline
 0 to 14~\newline
 0 is fine for ships, explosions, projectiles, and bunkers \\
\hline
{\em color} & Pixel color, one of\+: SSD1306\+\_\+\+BLACK, SSD1306\+\_\+\+WHITE or SSD1306\+\_\+\+INVERSE. \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
None (void). 
\end{DoxyReturn}
\begin{DoxyNote}{Note}
Bitmaps defined above were created for the LM3\+S1968 or LM3\+S8962\textquotesingle{}s 4-\/bit grayscale OLED display. More recently they are created using Microsoft Paint (or your favorite drawing program), saved as a 16-\/color bitmap, and converted from binary to text using Bmp\+Convert.\+exe. Then copy and paste this text into the program, which will create a constant containing the image that will be loaded into ROM at compile time. These images will still contain their header data and may contain padding to preserve 4-\/byte alignment. This function takes a bitmap in the previously described format and puts its image data in the proper location in the buffer so the image will appear on the screen after the next call to~\newline
 SSD1306\+\_\+\+Display\+Buffer();~\newline
 The interface and operation of this process is modeled after RIT128x96x4\+\_\+\+BMP(x, y, image); 
\end{DoxyNote}
